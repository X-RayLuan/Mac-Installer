# 키워드 리서처 스킬

## 역할

사용자가 제공한 주제/키워드를 기반으로 웹 검색을 수행하여 네이버 블로그에 최적화된 키워드 전략을 수립합니다.

## 실행 절차

### 1단계: 시드 키워드 확장

사용자의 입력 키워드를 시드로 삼아 웹 검색을 수행합니다.

검색 쿼리 패턴:

- `{키워드} 블로그` — 블로그 콘텐츠 트렌드 파악
- `{키워드} 추천 2026` — 최신 트렌드 확인
- `{키워드} 관련 키워드` — 연관 키워드 수집
- `네이버 {키워드} 검색량` — 검색 볼륨 추정

### 2단계: 키워드 분류

수집된 키워드를 아래 기준으로 분류합니다:

```
메인 키워드 (1개): 검색량 가장 높고 글의 핵심 주제를 대표
서브 키워드 (3~5개): 메인 키워드와 밀접하게 관련된 보조 키워드
롱테일 키워드 (5~10개): 구체적이고 경쟁이 낮은 세부 키워드
```

### 3단계: 검색 의도 분류

`knowledge/naver-seo-rules.md`의 검색 의도 분류표를 참조하여 메인 키워드의 검색 의도를 판별합니다:

- **정보 탐색**: ~하는 방법, ~하는 법, ~이란
- **비교/선택**: 추천, BEST, TOP, 순위
- **체험 확인**: 후기, 리뷰, 사용기
- **비교 분석**: vs, 비교, 차이
- **구매 결정**: 가격, 할인, 구매

### 4단계: 경쟁 강도 평가

웹 검색 결과를 기반으로 간략한 경쟁 분석을 수행합니다:

- 상위 결과에 대형 블로그/공식 사이트가 많으면 → 경쟁 높음
- 개인 블로그가 상위에 있으면 → 진입 가능
- 최근 글이 적으면 → 블루오션 가능성

### 5단계: 결과 출력

```
## 🔍 키워드 리서치 결과

### 메인 키워드
- **{키워드}**: 검색 의도 — {의도 유형}

### 서브 키워드
1. {서브 키워드 1} — {검색 의도}
2. {서브 키워드 2} — {검색 의도}
3. {서브 키워드 3} — {검색 의도}

### 롱테일 키워드
- {롱테일 1}, {롱테일 2}, {롱테일 3}, ...

### 추천 글 유형: {유형}
### 경쟁 강도: {높음/중간/낮음}

이 키워드로 진행할까요? 변경하고 싶은 키워드가 있으면 말씀해주세요.
```

## 주의사항

- 검색량 데이터는 정확한 수치가 아닌 상대적 추정치임을 사용자에게 안내
- 키워드 확정 전 반드시 사용자 확인을 받을 것
- 너무 경쟁이 높은 키워드만 추천하지 말고, 실현 가능한 롱테일도 함께 제안
