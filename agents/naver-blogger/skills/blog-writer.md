# 블로그 라이터 스킬

## 역할

키워드 리서치와 경쟁 분석 결과를 기반으로 네이버 블로그에 최적화된 글을 작성합니다.

## 입력 요건

이 스킬을 실행하기 전 아래 정보가 준비되어야 합니다:

- 메인 키워드 + 서브 키워드 (keyword-researcher 결과)
- 경쟁 분석 인사이트 (competitor-analyzer 결과)
- 글 유형 및 해당 템플릿 (instructions.md에서 선택)

## 작성 절차

### 1단계: 제목 작성

`knowledge/keyword-patterns.md`의 제목 패턴을 참조합니다.

규칙:

- 메인 키워드를 제목 앞쪽에 배치
- 30자 이내
- 클릭을 유도하는 수식어 추가 (구체적 수치, 경험 표현)
- **제목 후보 3개를 생성하고 최적안을 선택**

### 2단계: 아웃라인 구성

선택된 템플릿(`templates/` 디렉토리)의 구조를 따릅니다.

규칙:

- 소제목에 서브 키워드 분산 배치
- 소제목만 읽어도 글 내용을 파악할 수 있게
- 각 섹션의 핵심 포인트를 미리 정리

### 3단계: 본문 작성

`knowledge/formatting-guide.md`의 규칙을 준수합니다.

**도입부** (100~200자):

- 독자의 공감/문제 제기로 시작
- 메인 키워드 첫 100자 내 포함
- "이 글에서 ~를 정리했습니다" 식의 예고

**각 섹션** (200~400자):

- 소제목으로 시작
- 구체적 정보 포함 (수치, 이름, 위치 등)
- 개인 경험 기반 서술 ("직접 써보니~", "방문해보니~")
- 300자마다 `📸 [이미지 설명]` 마커 삽입

**마무리** (100~200자):

- 핵심 내용 1~2줄 요약
- CTA(행동 유도) 포함

### 4단계: 키워드 밀도 조정

`knowledge/keyword-patterns.md`의 밀도 공식을 적용합니다.

- 메인 키워드: 전체 글 대비 2~3% 밀도
- 서브 키워드: 각 2~3회 자연스럽게 분산
- 부자연스러운 키워드 삽입은 절대 금지

### 5단계: 최종 검수

작성 완료 후 자체 검수합니다:

- [ ] 메인 키워드가 제목 앞쪽에 있는가?
- [ ] 첫 100자에 메인 키워드가 포함되었는가?
- [ ] 소제목이 3개 이상인가?
- [ ] 이미지 마커가 300자마다 있는가?
- [ ] 마무리에 CTA가 있는가?
- [ ] 전체 글자 수가 1,500~3,000자 범위인가?
- [ ] AI가 쓴 티가 나지 않는 자연스러운 문체인가?

## 문체 규칙

### 자연스러운 한국어 작성 원칙

```
1. 다양한 문장 길이: 짧은 문장과 긴 문장을 섞어 리듬감 생성
2. 구어체 혼합: "사실 이게 핵심인데요" 같은 구어적 표현 적절히 혼합
3. 접속사 변화: "그리고" 반복 대신 "또한", "게다가", "특히", "무엇보다" 등 다양하게
4. 능동태 우선: "~되었습니다"보다 "~했습니다"
5. 구체적 표현: "좋았다"보다 "가성비가 뛰어났다", "인테리어가 깔끔했다"
```

### 금지 패턴

```
❌ "오늘은 ~에 대해 알아보겠습니다" (전형적 AI 도입부)
❌ "결론적으로 말하자면" (AI 특유의 결론 패턴)
❌ "~할 수 있습니다" 반복 (단조로운 어미)
❌ 같은 접속사 3회 이상 연속 사용
❌ 한 문단에 같은 단어 3회 이상 반복
```
