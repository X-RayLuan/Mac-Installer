# 비교형 (Comparison) 템플릿

## 적합한 키워드

"비교", "vs", "차이", "뭐가 나을까", "어떤 게 좋을까"

## 글 구조

```
[제목]: {A} vs {B} + {비교 핵심 축}
예: "갤럭시 S25 vs 아이폰 16 카메라 성능 비교 실사용 후기"

---

[도입부] (100~200자)
- "A와 B 중 고민되시나요?"
- 두 제품/서비스를 모두 사용해본 경험 강조
- "핵심 차이를 {N}가지로 비교했습니다"
- 메인 키워드 포함

📸 [A와 B를 나란히 놓은 비교 이미지]

---

[소제목 1]: 한눈에 비교표
| 항목 | {A} | {B} |
|------|-----|-----|
| 가격 | ... | ... |
| 핵심 스펙 | ... | ... |
| ... | ... | ... |

- 표를 먼저 보여주면 체류 시간 증가
- 핵심 차이에 굵은 글씨 표시

---

[소제목 2]: {비교 축 1} — {핵심 차이 요약}
- 객관적 비교 + 주관적 경험
- "실제로 사용해보면~"
- 승자 명시 (명확한 가치 제공)

📸 [비교 축 1 관련 이미지]

---

[소제목 3~N]: 동일 구조로 비교 축 반복
- 비교 축마다 승패 명시
- 구체적 수치/경험 근거 포함

📸 [각 비교 축 이미지]

---

[소제목: 결론 — 이런 사람은 A, 이런 사람은 B]
- 무조건 한쪽 추천 대신 상황별 추천
- "~가 중요하다면 A"
- "~를 더 원한다면 B"
- 가장 범용적 추천도 명시 ("대부분의 경우라면~")

---

[마무리] (100~150자)
- "결론적으로 저는 ~를 선택했습니다" (개인 선택 공개)
- "고민되시면 댓글로~" CTA
```

## 특징

- 비교 축은 3~5개가 적당 (너무 많으면 산만)
- 각 비교 축에서 "어느 쪽이 나은지" 명확히 말해야 함 (애매하면 가치 없음)
- 공정한 비교 톤 유지 (한쪽 편들기 X) → 최종 선택만 개인 의견으로
- 비교표를 글 초반에 놓으면 전체 그림을 빠르게 파악 → 이후 본문 정독 유도
